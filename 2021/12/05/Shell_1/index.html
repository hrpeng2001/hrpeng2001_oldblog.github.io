一、Shell简介与脚本运行

  Shell即“命令解释器”，是用户同计算机内核交互的桥梁；包括图形界面的Shell和命令行界面的Shell。在本文中，关注的是Linux系统下命令行界面的Shell，在Linux系统中，Shell既是一种命令语言，又是程序设计语言。
  Linux下，有多种Shell，包括Bourne Shell(/usr/bin/sh 或 bin/sh)、Bourne Again Shell(/bin/bash)、C Shell(/usr/bin/csh)、K Shell(/usr/bin/ksh)、Shell of Root(/sbin/sh)。关于各种Shell的差别，可以参考：https://en.wikipedia.org/wiki/Comparison_of_command_shells
  注意：本教程大纲以bash为例进行介绍。

Shell脚本，即使用Shell编写的脚本程序，运行脚本具有两种方法：
（1）作为可执行程序（该方式在脚本文件内部需要在首行以“#！”标明解释器）

（2）作为解释器参数

二、Shell变量

2.1 变量的命名规则

  • shell的变量命名规则同其他编程语言是类似的，包括以下规则：
  • 命名只能使用英文字母，数字和下划线，首个字符不能以数字开头。
  • 中间不能有空格
  • 不能使用下划线以外的标点符号
  • 不能使用bash的关键字

2.2 变量的引用、属性

  在bash对变量进行赋值时，不需要加美元符号“$”；但对变量进行引用时，需要加美元符号“$”，并建议使用大括号“{ }”包住变量，以表示变量的边界。
  声明变量后，使用readonly可指定其为只读变量，使用unset可以删除变量。

2.3 变量的类型

  shell程序的变量从作用范围上分，可以分为局部变量、环境变量、和shell变量；从数据类型上分，则主要有字符串变量和数值变量。

2.3.1 字符串变量

字符串，可以用单引号、双引号、或不用引号；但单引号中变量是无效的，转义字符也是无效的。

2.4 Shell注释

三、Shell传递参数

  无论对于函数还是脚本，都以在文件或函数名后空格加参数的方式传递参数。

  在脚本或函数内，则以$n的形式获取参数。
此外，有几个特殊字符用于处理参数：

  • $# 传递到脚本的参数个数
  • $* 以当个字符串显示所有向脚本传递的参数
  • $$ 脚本运行的当前进程ID号
  • $! 后台运行的最后一个进程的ID号
  • $@ 与$*相似，但是逐个返回这些参数
  • $- 显示Shell使用的当前选项
  • $? 显示最后命令的退出状态

  Shell支持多种运算符，包括：算数、关系、布尔、字符串、文件测试运算符。
  原生bash不支持简单的数学运算，但是可以通过其它命令实现，例如awk和expr

  • 算术运算符（“+”“-”“*”“/”“%”“=”“==”“!=”）
  • 关系运算符（“-eq”“-ne”“-gt”“-lt”“-ge”“-le”）
  • 布尔运算符（“！”“-o”“-a”）
  • 逻辑运算符 （“&&”“||”）和布尔运算符差不多，但要两个中括号包裹

八、输入和输出的重定向

8.1 重定向方法

UNIX的默认的标准输入输出都是终端，重定向则通常到文件。
重定向列表如下：

  • command > file 输出重定向到file
  • command < file 输入重定向到file
  • command >> file 将输出以追加的方式重定向到file
  • n > file 将文件描述符为n的文件重定向到file
  • n >> file 将文件描述符为n的文件以追加的方式重定向到file
  • n >& m 将输出文件m和n合并
  • n <& m 将输入文件m和n合并
  • <<tag 将开始标记tag和结束标记tag之间的内容作为输入
